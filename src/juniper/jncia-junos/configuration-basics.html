This objective is about getting a device into a usable state and protecting that configuration once it exists. Focus on the commit workflow, user access, and a few standard system services.

What to know

  • Factory-default devices need basic setup before they can be managed on the network.
  • Initial configuration usually includes hostname, root authentication, management addressing, and a default route if needed.
  • User accounts define who can log in. Login classes define what those users are allowed to do.
  • Junos supports local authentication and centralized methods such as RADIUS and TACACS+.
  • Interfaces are configured under the `interfaces` hierarchy with unit numbers, families, and addresses.
  • Configuration groups reduce repetition by applying inherited settings to multiple places in the configuration.
  • Common base services include NTP for time, SNMP for monitoring, and syslog for event logging.
  • Archival and rescue configuration features help with recovery. A rescue configuration is a known-good checkpoint.
  • Tracing and logging help troubleshoot both normal behavior and failures.

Quick checks

  • You must define the root password during initial setup before you can commit.
  • Login class controls permissions; user account ties a person or template to that access level.
  • Use rescue config for fast recovery to a known-good state.
