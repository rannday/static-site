The CLI is the primary interface to know for the exam. J-Web matters at a high level, but most objective coverage here is about moving around the CLI and understanding the Junos commit model.

What to know

  • Operational mode is for monitoring and troubleshooting. Configuration mode is for editing the candidate configuration.
  • The active configuration is what the device is currently running. The candidate configuration is the working copy you edit before `commit`.
  • Common CLI flow: enter `configure`, make changes, review with `show` or `show | compare`, then `commit`.
  • CLI help uses `?`, tab completion, and context-sensitive prompts.
  • Output filtering is commonly done with pipes such as `match`, `except`, `count`, or `display set`.
  • Rollback lets you return to previously committed configurations. `rollback 0` discards uncommitted candidate changes.
  • Configuration files can be viewed, compared, loaded, and saved from the CLI. Junos keeps multiple rollback files automatically.
  • J-Web is the browser-based Junos interface used for initial setup, basic management, and troubleshooting.

Common commands

  • `show interfaces terse`
  • `show configuration`
  • `show | compare`
  • `rollback 1`
  • `load set`, `load merge`, `save`, `commit`
