Routing questions at this level are about basic forwarding logic, route selection, and where different route types live in Junos. Keep the routing table versus forwarding table distinction clear.

What to know

  • The routing table stores learned and configured routes. The forwarding table contains the active routes used to forward packets.
  • Traffic forwarding is based on destination lookup and longest-prefix match.
  • Junos maintains default tables such as `inet.0` for IPv4 unicast routes, and additional tables can exist for other families or routing instances.
  • Route preference is Junos administrative preference. Lower preference is better.
  • Static routes are manually configured and are often used for simple reachability, default routes, or backup paths.
  • Routing instances separate interfaces, routing tables, and protocol state. They are used for traffic separation and virtualization.
  • Dynamic routing protocols automatically learn and exchange routes. Their main advantage is adaptability and scalability compared with static-only designs.

Quick checks

  • Routing table decides the best route; forwarding table is what the PFE actually uses.
  • Lower route preference wins.
  • `inet.0` is the normal default IPv4 unicast table on Junos.
