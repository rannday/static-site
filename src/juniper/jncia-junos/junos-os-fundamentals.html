Junos separates control functions from packet forwarding. That separation is a core exam theme because it explains how routing, management, and traffic handling fit together.

What to know

  • The Routing Engine (RE) handles the control plane: routing protocols, system management, configuration, and user access.
  • The Packet Forwarding Engine (PFE) handles the forwarding plane: Layer 2 switching, route lookups, and packet forwarding in hardware.
  • The RE builds routing tables, picks active routes, and installs the forwarding table that the PFE uses.
  • Transit traffic is normal user traffic being forwarded through the device. It should stay in the forwarding path and avoid unnecessary RE processing.
  • Exception traffic is traffic that must go to the RE, such as routing protocol packets, management traffic, or packets that need special handling.
  • Junos is modular, so failures in one process are less likely to take down the entire system.

Quick checks

  • Control plane = RE.
  • Forwarding plane = PFE.
  • Active routes come from the routing table; packet forwarding uses the forwarding table.
