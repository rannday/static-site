This section is mostly operational-mode awareness: know which commands you would use, what they tell you, and the basic maintenance workflow for software and recovery tasks.

What to know

  • `show` commands display status, routing, interfaces, alarms, and configuration details.
  • `monitor` commands give live updates, such as real-time interface counters or traffic views.
  • Interface monitoring should include traffic rates, packet counters, drops, and physical errors.
  • Basic network tools include `ping`, `traceroute`, `telnet`, and `ssh` for reachability and path testing.
  • Upgrades require planning: back up the configuration, validate the package, install it, and reboot into the new version.
  • Power operations use operational commands, not abrupt power removal. Know normal shutdown and reboot concepts.
  • Root password recovery is a standard break-glass procedure and is part of basic platform administration.

Common commands

  • `show interfaces terse`
  • `show interfaces extensive`
  • `show system alarms`
  • `monitor interface traffic`
  • `ping`, `traceroute`, `ssh`, `telnet`
